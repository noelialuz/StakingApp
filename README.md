### Architectural Decisions & Design Patterns

* **CEI Pattern (Checks-Effects-Interactions):** Strictly applied across all state-modifying functions (e.g., `stake`, `withdraw`, `claimRewards`) to inherently eliminate reentrancy vectors before external calls are executed.
* **Gas-Optimized Custom Errors:** Completely decoupled from legacy string-based `require` statements. Reverts utilize custom error selectors (e.g., `error InsufficientBalance()`), drastically lowering deployment and execution bytecode footprints.
* **Access Control & Safeguards:** Inherits OpenZeppelinвЂ™s `Ownable` for restricted administrative parameters and implements explicit state tracking over raw user balances to maintain mathematical invariants.

## рџ“ђ Core Protocol Specifications

The protocol allows users to lock an accepted ERC-20 token to accumulate reward tokens calculated dynamically over time.

### Mathematical Invariants & State Syncing
The rewards engine calculates yield linearly based on time elapsed and total capital staked. 

$$Reward = \int_{t_0}^{t_1} (StakedAmount \times RewardRate) \, dt$$

The codebase implements checkpointing techniques to avoid $O(N)$ loops, ensuring that rewards are updated lazily on every user interaction (`stake`, `withdraw`), keeping the execution cost at $O(1)$ complexity.

## рџ”’ Security & Test Coverage Matrix

Quality assurance is driven by a test-suite designed from an auditorвЂ™s perspective, prioritizing boundary conditions, front-running mitigation, and extreme state changes.

### 1. Unit Testing (Deterministic Verification)
* Comprehensive coverage of "happy paths" and "unhappy paths".
* State-forking and execution isolation using Foundry cheatcodes (`vm.prank`, `vm.expectEmit`).
* Strict validation of Custom Error selectors and internal data encodings via `abi.encodeWithSelector`.

### 2. Property-Based Testing (Fuzzing)
Instead of static variables, the suite runs thousands of test cases per function with semi-randomized inputs generated by the EVM.
* **Bounded Fuzzing:** Inputs are mathematically restricted via `bound()` to valid economic thresholds, eliminating noise from unrealistic extreme numbers (e.g., values exceeding total token supply).
* **State Invariant Fuzzing:** Testing that the contract's internal accounting of total staked tokens always matches the physical `balanceOf(address(this))` of the ERC-20 token.
